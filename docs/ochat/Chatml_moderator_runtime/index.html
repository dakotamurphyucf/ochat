Module Chatml_moderator_runtime

Host-side session runtime for ChatML moderator scripts.

This module sits above the minimal ChatML core and provides the session-oriented runtime described in the moderator design:

  • scripts are compiled once and instantiated per session,
  • on_event returns Chatml_lang.task values,
  • the host interprets those tasks via an operation registry,
  • local transactional effects are buffered and committed on success.

The runtime is intentionally generic: embedders provide concrete handlers for tool/model/scheduling integration while the module supplies the common execution model.

module Builtin_surface = Chatml.Chatml_builtin_surface
type compiled_script

Compiled script artifact produced by compile_script.

type session

Per-session runtime instance holding durable state and execution queues.

type log_level =
  1. | Debug
  2. | Info
  3. | Warn
  4. | Error_level

Diagnostic log level used by the default Log.* operation family.

type turn_effect =
  1. | Prepend_system of string
  2. | Append_message of Chatml.Chatml_lang.value
  3. | Replace_message of string * Chatml.Chatml_lang.value
  4. | Delete_message of string
  5. | Halt of string

Structured local turn mutations used by the default Turn.* handlers.

The public ChatML moderator surface prefers item-oriented names such as Turn.append_item, Turn.replace_item, and Turn.delete_item. The legacy *_message builtin names remain as aliases and decode to these same runtime constructors.

type tool_moderation =
  1. | Approve
  2. | Reject of string
  3. | Rewrite_args of Chatml.Chatml_lang.value
  4. | Redirect of string * Chatml.Chatml_lang.value

Structured tool-moderation actions used by the default Tool.* moderation handlers.

type local_effect =
  1. | Turn_effect of turn_effect
  2. | Tool_moderation_effect of tool_moderation
  3. | Emit_internal_event of Chatml.Chatml_lang.value
  4. | Request_compaction
  5. | Request_turn
  6. | End_session of string

Structured local effects that can be decoded from committed transactional operations after handle_event succeeds.

type op_kind =
  1. | Local_transactional
  2. | External_sync
  3. | External_async
  4. | Diagnostic

Runtime classification of task operations.

type op_def = {
  1. name : string;
  2. kind : op_kind;
  3. perform : session -> Chatml.Chatml_lang.value list -> (Chatml.Chatml_lang.value, string) Stdlib.result;
  4. phase_check : string -> (unit, string) Stdlib.result;
}

Definition of a host operation exposed to ChatML tasks.

type runtime_config = {
  1. surface : Builtin_surface.surface;
  2. operations : op_def list;
}

Runtime configuration shared across sessions for a given host/runtime embedding.

type compiled_entrypoints = {
  1. initial_state_name : string;
  2. on_event_name : string;
}

Names of the convention-based script entrypoints.

type default_handlers = {
  1. on_log : session -> level:log_level -> message:string -> (unit, string) Stdlib.result;
  2. on_turn_effect : session -> turn_effect -> (unit, string) Stdlib.result;
  3. on_tool_moderation : session -> tool_moderation -> (unit, string) Stdlib.result;
  4. on_tool_call : session -> name:string -> args:Chatml.Chatml_lang.value -> (Chatml.Chatml_lang.value, string) Stdlib.result;
  5. on_tool_spawn : session -> name:string -> args:Chatml.Chatml_lang.value -> (string, string) Stdlib.result;
  6. on_model_call : session -> recipe:string -> payload:Chatml.Chatml_lang.value -> (Chatml.Chatml_lang.value, string) Stdlib.result;
  7. on_model_spawn : session -> recipe:string -> payload:Chatml.Chatml_lang.value -> (string, string) Stdlib.result;
  8. on_schedule_after_ms : session -> delay_ms:int -> payload:Chatml.Chatml_lang.value -> (string, string) Stdlib.result;
  9. on_schedule_cancel : session -> id:string -> (unit, string) Stdlib.result;
  10. on_request_compaction : session -> (unit, string) Stdlib.result;
  11. on_end_session : session -> reason:string -> (unit, string) Stdlib.result;
  12. on_request_turn : session -> (unit, string) Stdlib.result;
}

Host callbacks used to build the standard moderator operation registry. Handlers that are left at their defaults either act as no-ops for local/diagnostic behavior or return a clear "... is not configured" error for external integrations.

val string_of_log_level : log_level -> string

Render a log level using the names expected by human-facing diagnostics.

val allow_all_phases : string -> (unit, string) Stdlib.result

Phase check that accepts every phase.

val require_phases : string list -> string -> (unit, string) Stdlib.result

Build a phase check that accepts only the listed phase names.

val default_handlers : default_handlers

Default handler bundle used by default_operations.

val default_operations : ?handlers:default_handlers -> unit -> op_def list

Construct the standard operation registry for moderator runtimes from a bundle of host callbacks.

val default_runtime_config : ?surface:Builtin_surface.surface -> ?handlers:default_handlers -> unit -> runtime_config

Convenience constructor for a runtime configuration using the standard moderator surface and default operation registry.

val compile_script : ?surface:Builtin_surface.surface -> source:string -> unit -> (compiled_script, string) Stdlib.result

Parse, typecheck, and resolve a script once. The default surface is Builtin_surface.moderator_surface.

val instantiate_session : runtime_config -> compiled_script -> entrypoints:compiled_entrypoints -> (session, string) Stdlib.result

Instantiate a compiled script in a fresh per-session environment and load the configured entrypoints.

val current_state : session -> Chatml.Chatml_lang.value

Current durable script state for the session.

val current_phase : session -> string option

Current phase while a handler is actively running, if any.

val pending_local_effects : session -> Chatml.Chatml_lang.eff list

Local transactional effects buffered during the current handler execution.

val committed_local_effects : session -> Chatml.Chatml_lang.eff list

All committed local transactional effects observed so far for the session, in execution order.

val decode_local_effect : Chatml.Chatml_lang.eff -> (local_effect, string) Stdlib.result

Decode one committed transactional effect into the structured local runtime vocabulary.

val decode_local_effects : Chatml.Chatml_lang.eff list -> (local_effect list, string) Stdlib.result

Decode committed transactional effects in execution order. This is the preferred way for embedders to inspect local turn mutations, tool moderation actions, and runtime requests because it reflects the runtime's transactional commit semantics.

val queued_events : session -> Chatml.Chatml_lang.value list

Buffered internal events currently queued for later delivery.

val take_queued_event : session -> Chatml.Chatml_lang.value option

Remove and return the oldest queued internal event, if one exists.

val restore : session -> state:Chatml.Chatml_lang.value -> queued_events:Chatml.Chatml_lang.value list -> halted:bool -> (unit, string) Stdlib.result

Replace the session's durable runtime state from a persisted snapshot.

Any committed local effects from prior runs are cleared.

val is_halted : session -> bool

Whether the session has been ended by a committed runtime action.

val emit_internal_event : session -> Chatml.Chatml_lang.value -> (unit, string) Stdlib.result

Append an internal event to the current handler's transactional output buffer. This is primarily useful for custom host operations.

val request_session_end : session -> reason:string -> (unit, string) Stdlib.result

Request session termination from within the current handler's transactional context.

val handle_event : session -> context:Chatml.Chatml_lang.value -> event:Chatml.Chatml_lang.value -> (unit, string) Stdlib.result

Handle one event:

  • invokes on_event,
  • interprets the returned task,
  • commits buffered state/effects on success,
  • or rolls back local transactional buffers on failure.
val enqueue_internal_event : session -> Chatml.Chatml_lang.value -> (unit, string) Stdlib.result

Enqueue an internal event for later delivery via the host's internal-event replay mechanism. Unlike Runtime.emit, this is host-driven and does not require an active handler execution.
