Module Chat_response.Cache

In-memory and on-disk cache for agent executions.

The Cache module provides a small **TTL-based LRU cache** that maps an Prompt.Chat_markdown.agent_content value â€“ effectively the *complete specification* of an `<agent/>` element (its URL and any inline user items) â€“ to the textual response returned by running that agent.

The cache lives entirely in memory while the program executes but can be serialised to / read from disk using save and load. Persistence is binary, deterministic, and compact thanks to Bin_prot.

Design

â€¢ LRU policy with a *time-to-live* (TTL) per entry â€“ implemented by Ttl_lru_cache. â€¢ Key is Prompt.Chat_markdown.agent_content. â€¢ Value is the **verbatim assistant answer** (a string).

High-level API

  • create constructs an empty cache with a bounded size.
  • find_or_add returns a cached value or computes and stores a fresh one.
  • load/save transparently handle on-disk persistence between runs.

Example: reading a cache at start-up and persisting it on exit

  let cache_file = Path.(cwd / "cache.bin") in
  let cache = Cache.load ~file:cache_file ~max_size:1000 () in

  (* â€¦ use [cache] during the program â€¦ *)

  Cache.save ~file:cache_file cache

val create : max_size:int -> unit -> 'a LRU.t

create ~max_size () returns a fresh in-memory cache that can hold at most max_size entries. When inserting a new item and the cache is full, the *least recently used* entry is evicted.

Entries are still subject to their individual TTL even if the cache is not full.

val to_persistent : string LRU.t -> persistent_form

Internal: convert an in-memory cache into the serialisable form that is written on disk. Exposed only for unit tests.

val of_persistent : persistent_form -> string LRU.t

Internal: inverse of to_persistent.

val write_file : file:Bin_prot_utils_eio.path -> string LRU.t -> unit

write_file ~file cache serialises cache using Bin_prot and writes it atomically to file. The function truncates any existing file.

val read_file : file:Bin_prot_utils_eio.path -> string LRU.t

read_file ~file loads a cache previously written with write_file.

val find_or_add : 'a LRU.t -> Key.t -> ttl:Core.Time_ns.Span.t -> default:(unit -> 'a) -> 'a

find_or_add t key ~ttl ~default returns the value associated with key if it is present *and* younger than ttl. Otherwise it evaluates default (), stores the result under key together with the supplied ttl, and returns it.

This wrapper is a direct re-export of Ttl_lru_cache.find_or_add so that callers do not need to reference the underlying implementation module.

val load : file:Eio.Fs.dir_ty Eio.Path.t -> max_size:int -> unit -> string LRU.t

load ~file ~max_size () returns a cache backed by the on-disk file file. If the file does not exist the function returns a fresh cache built with create~max_size.

val save : file:Bin_prot_utils_eio.path -> string LRU.t -> unit

save ~file cache writes cache to disk using write_file.
