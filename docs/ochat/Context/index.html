Module Context

module Model = Openai.Responses.Request

Immutable context record threaded through the meta-prompting pipeline to avoid global mutable state. Additional fields can be appended in future tasks without changing call-sites thanks to the optional record update syntax.

type action =
  1. | Generate
  2. | Update
type t = {
  1. proposer_model : Model.model option;
  2. rng : Core.Random.State.t;
  3. env : Eio_unix.Stdenv.base option;
  4. guidelines : string option;
  5. model_to_optimize : Model.model option;
  6. action : action;
}

t is the type of the context record. It contains a random state for reproducibility, an optional proposer model override, an optional environment for Eio, and optional guidelines for the meta-prompting process.

val default : unit -> t

default () builds a fresh context with a newly initialised random state and no proposer model override. Suitable for backward- compatibility when callers have not yet adopted the explicit context API.

val with_proposer_model : t -> model:Model.model option -> t

with_proposer_model ctx ~model returns a shallow copy of ctx that overrides the proposer_model field with model. This helper avoids record-update boilerplate at call sites.

val with_guidelines : t -> guidelines:string option -> t

with_guidelines ctx ~guidelines returns a shallow copy of ctx with the guidelines field replaced by guidelines.
