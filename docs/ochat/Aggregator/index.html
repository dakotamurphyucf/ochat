Module Aggregator

Aggregation strategies for combining a list of judge scores into a single scalar. The module provides a small DSL where an aggregator is just a function of type float list -> float. Convenience constructors for common strategies are provided below.

type t = float list -> float

The abstract type of an aggregator â€“ a function that maps a (possibly empty) list of scores to a single scalar. Implementations must be total: they should accept the empty list and return a sensible default value (conventionally 0.0).

val mean : t

Arithmetic mean of the input scores. Returns 0.0 for an empty list.

val median : t

Median of the input scores (50th percentile). For an even number of elements the arithmetic mean of the two middle values is returned. Returns 0.0 for an empty list.

val trimmed_mean : trim:float -> t

trimmed_mean ~trim discards the bottom trim fraction and the top trim fraction of the sorted scores before computing the mean. The trim parameter must satisfy 0.0 <= trim < 0.5. For instance, trim = 0.1 yields a 10 % trimmed mean (10 % of the observations at each tail are removed). If the resulting set is empty the function returns 0.0.

val weighted : weights:float list -> t

weighted ~weights returns an aggregator that computes the weighted arithmetic mean using the supplied weights. The weight list must have the same length as the list of scores it is later applied to. If the two lengths differ the function falls back to the unweighted mean to avoid accidental mis-alignment. Negative weights are allowed but discouraged. If the sum of weights is zero the result is 0.0.

val min : t

Minimum of the input scores (or 0.0 if the list is empty).

val max : t

Maximum of the input scores (or 0.0 if the list is empty).
