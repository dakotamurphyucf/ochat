Module App_streaming.Context

start ctx ~history ~op_id runs a single OpenAI streaming request and reports progress to the internal event stream.

The worker emits:

  • `Streaming_started once a dedicated streaming switch exists;
  • `Stream and `Stream_batch events for incremental deltas;
  • `Tool_output items for tool call outputs;
  • `Streaming_done with the final item list; or
  • `Streaming_error on failure or cancellation.

The function catches all exceptions and converts them into a `Streaming_error event.

All inputs other than history and op_id are bundled in Context.t.

  • parameter history

    OpenAI item history that seeds the request.

  • parameter op_id

    Tags events so the reducer can ignore stale messages.

Example:

  let streams : Chat_tui.App_context.Streams.t =
    { input; internal; system }
  in
  let services : Chat_tui.App_context.Services.t =
    { env; ui_sw; cwd; cache; datadir; session }
  in
  let resources : Chat_tui.App_context.Resources.t = { services; streams; ui } in
  let ctx : Chat_tui.App_streaming.Context.t =
    { shared = resources
    ; cfg
    ; tools
    ; tool_tbl
    ; parallel_tool_calls = true
    ; history_compaction = true
    }
  in
  Chat_tui.App_streaming.start ctx ~history ~op_id:0
type t = {
  1. shared : App_context.Resources.t;
  2. cfg : Chat_response.Config.t;
  3. tools : Openai.Responses.Request.Tool.t list;
  4. tool_tbl : (string, string -> Openai.Responses.Tool_output.Output.t) Core.Hashtbl.t;
  5. parallel_tool_calls : bool;
  6. history_compaction : bool;
}
