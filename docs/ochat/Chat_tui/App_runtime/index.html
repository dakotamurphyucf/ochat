Module Chat_tui.App_runtime

Mutable runtime state for Chat_tui.App's event loop.

This module defines the runtime state record that is threaded through Chat_tui.App_reducer.run. It captures:

  • The current in-flight operation (streaming or compaction), if any.
  • A FIFO queue of user actions that are deferred while an operation runs.
  • Cancellation requests that arrive while an operation is still "starting" (before the worker fibre has published its Eio.Switch.t).

The types are intentionally low-level because multiple helper modules coordinate via mutable fields. Treat this module as internal plumbing; it is exposed primarily to keep Chat_tui.App small and to support white-box tests of the event loop.

type op =
  1. | Streaming of {
    1. sw : Eio.Switch.t;
    2. id : int;
    }
    (*

    An assistant response is currently streaming.

    The operation can be cancelled by failing sw.

    *)
  2. | Compacting of {
    1. sw : Eio.Switch.t;
    2. id : int;
    }
    (*

    History compaction is currently running.

    *)
  3. | Starting_streaming of {
    1. id : int;
    }
    (*

    A streaming worker has been forked but has not yet published its switch.

    *)
  4. | Starting_compaction of {
    1. id : int;
    }
    (*

    A compaction worker has been forked but has not yet published its switch.

    *)

A currently running (or about-to-run) background operation.

type typeahead_op =
  1. | Typeahead of {
    1. sw : Eio.Switch.t;
    2. id : int;
    }
    (*

    A type-ahead worker is currently running.

    The operation can be cancelled by failing sw.

    *)
  2. | Starting_typeahead of {
    1. id : int;
    }
    (*

    A type-ahead worker has been forked but has not yet published its switch.

    *)

Tracking state for background type-ahead completion work.

type submit_request = {
  1. text : string;
  2. draft_mode : Model.draft_mode;
}

A snapshot of the editor state that is submitted to the assistant.

type queued_action =
  1. | Submit of submit_request
  2. | Compact

Work that was requested while another operation is running.

type t = {
  1. model : Model.t;
  2. mutable op : op option;
  3. mutable typeahead_op : typeahead_op option;
  4. pending : queued_action Core.Queue.t;
  5. quit_via_esc : bool Stdlib.ref;
    (*

    true iff the user hit Esc while idle, causing the app to exit.

    *)
  6. mutable next_op_id : int;
    (*

    Monotonically increasing operation id used to tag events.

    *)
  7. mutable cancel_streaming_on_start : bool;
    (*

    Records a cancellation request that arrived while the state was Starting_streaming.

    *)
  8. mutable cancel_compaction_on_start : bool;
    (*

    Records a cancellation request that arrived while the state was Starting_compaction.

    *)
  9. mutable cancel_typeahead_on_start : bool;
    (*

    Records a cancellation request that arrived while the state was Starting_typeahead.

    *)
}

Runtime container used by the app reducer and its helper modules.

val create : model:Model.t -> t

create ~model creates a fresh runtime container for model.

  • parameter model

    Mutable UI model that the reducer will mutate.

The returned value starts with no active operation, an empty pending queue, and a fresh operation id counter.

Example:

  let runtime = Chat_tui.App_runtime.create ~model in
  ignore (Chat_tui.App_runtime.alloc_op_id runtime : int)
val alloc_op_id : t -> int

alloc_op_id t allocates a fresh operation id.

The returned id is used to tag internal events so that the reducer can ignore stale messages from previously cancelled operations.

  • parameter t

    Runtime container whose internal counter should advance.
