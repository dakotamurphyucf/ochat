Module Chat_tui.Types

Shared data types for the Ochat terminal UI.

This module centralises the few fundamental types that every sub-module of Chat_tui needs. Moving them into their own compilation unit avoids cyclic dependencies between the renderer, controller and model layers.

type role = string

Role of a chat message. Expected values are the strings mandated by the OpenAI API: "system", "user", "assistant", "function". The module does not validate the value.

type message = role * string

One chat message represented as (role, content).

type msg_buffer = {
  1. text : string Stdlib.ref;
    (*

    Mutable accumulator for the streaming text.

    *)
  2. index : int;
    (*

    Index into Chat_tui.Model.t.messages that will hold the final message once streaming completes.

    *)
}

Streaming-time buffer. While we receive deltas from the OpenAI API we accumulate partial output in text and remember the target index into the messages list so the UI can update incrementally.

type patch =
  1. | Ensure_buffer of {
    1. id : string;
    2. role : string;
    }
    (*

    Ensure the buffer id exists, creating an empty entry with the supplied role and adding a placeholder message when necessary.

    *)
  2. | Append_text of {
    1. id : string;
    2. role : string;
    3. text : string;
    }
    (*

    Append text to the buffer id, allocating it on first use and updating the corresponding entry in Chat_tui.Model.t.messages.

    *)
  3. | Set_function_name of {
    1. id : string;
    2. name : string;
    }
    (*

    Record the function name associated with streaming buffer id.

    *)
  4. | Set_function_output of {
    1. id : string;
    2. output : string;
    }
    (*

    Store the JSON output returned by the function call for id.

    *)
  5. | Update_reasoning_idx of {
    1. id : string;
    2. idx : int;
    }
    (*

    Update the last-seen reasoning summary index for buffer id.

    *)
  6. | Add_user_message of {
    1. text : string;
    }
    (*

    Insert the user's prompt text into the chat history.

    *)
  7. | Add_placeholder_message of {
    1. role : string;
    2. text : string;
    }
    (*

    Append the transient placeholder (role, text). The message is rendered only in the UI and is not persisted.

    *)
