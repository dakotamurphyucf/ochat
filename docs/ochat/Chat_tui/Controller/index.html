Module Chat_tui.Controller

Event controller for the Chat-TUI.

`Controller` is the single entry-point that the main event loop uses to translate a raw Notty.Unescape.event into an in-memory mutation of the Chat_tui.Model.t. All logic in the file is pure with respect to IO – it only edits the mutable record fields of the given model and returns a reaction value that tells the caller what to do next.

Key maps are split into three sub-states that mirror Vim semantics:

• Insert – free-form text editing (default) • Normal – modal navigation & message manipulation (see Chat_tui.Controller_normal) • Cmdline – ':' command prompt (see Chat_tui.Controller_cmdline)

`Controller.handle_key` is therefore a dispatcher: it selects a key-handler based on model.mode and forwards the event. Insert-mode is implemented locally in this compilation unit, whereas Normal- and Command-line modes live in their respective sub-modules.

Insert-mode shortcuts

The implementation provides a pragmatic subset of desktop-editor shortcuts. Selected examples:

  • Arrow keys, Home / End – caret movement
  • Meta+←/→ or Ctrl+←/→ – word-wise movement
  • Ctrl-A / Ctrl-E – beginning / end of line
  • Backspace – delete previous character
  • Ctrl-K / Ctrl-U / Ctrl-W – kill to EOL / BOL / previous word
  • Ctrl-Y – yank the last killed text
  • Meta+Enter – submit the current prompt (Submit_input)
  • PageUp / PageDown – scroll conversation history
  • ESC – cancel streaming or return to Normal mode

The set of bindings is intentionally conservative; unsupported keys are returned as Unhandled so that outer layers may implement fallbacks.

Reactions

All controller variants share the same reaction type (defined in Chat_tui.Controller_types):

| Redraw         – visible state changed; re-render the UI
| Submit_input   – draft is ready; send it to the assistant
| Cancel_or_quit – ESC; cancel streaming or quit if idle
| Quit           – immediate termination (Ctrl-C / 'q')
| Unhandled      – event not recognised; try other handlers

Performance: all operations run in O(length input) at worst (duplicate line); typical keystrokes are sub-millisecond.

type reaction = Controller_types.reaction =
  1. | Redraw
    (*

    The event modified the visible state – caller should refresh.

    *)
  2. | Submit_input
    (*

    User pressed Meta+Enter to submit the prompt.

    *)
  3. | Cancel_or_quit
    (*

    ESC – cancel running request or quit.

    *)
  4. | Quit
    (*

    Immediate quit (Ctrl-C / q).

    *)
  5. | Unhandled
    (*

    Controller didn’t deal with the event.

    *)
val handle_key : model:Model.t -> term:Notty_eio.Term.t -> Notty.Unescape.event -> reaction

handle_key ~model ~term ev dispatches ev according to model.mode and returns a reaction.

Side-effects are limited to in-place updates of the model; the function performs no network or file IO. The term argument is required for helpers that must query the current terminal geometry (e.g. for paging calculations).
