Module Chat_tui.Controller

Event controller for the Chat-TUI.

`Controller` is the single entry-point that the main event loop uses to translate a raw Notty.Unescape.event into an in-memory mutation of the Chat_tui.Model.t. All logic in the file is pure with respect to IO ‚Äì it only edits the mutable record fields of the given model and returns a reaction value that tells the caller what to do next.

Key maps are split into three sub-states that mirror Vim semantics:

‚Ä¢ Insert ‚Äì free-form text editing (default) ‚Ä¢ Normal ‚Äì modal navigation & message manipulation (see Chat_tui.Controller_normal) ‚Ä¢ Cmdline ‚Äì ':' command prompt (see Chat_tui.Controller_cmdline)

`Controller.handle_key` is therefore a dispatcher: it selects a key-handler based on model.mode and forwards the event. Insert-mode is implemented locally in this compilation unit, whereas Normal- and Command-line modes live in their respective sub-modules.

Insert-mode shortcuts

The implementation provides a pragmatic subset of desktop-editor shortcuts. Selected examples:

  • Left / Right arrow keys ‚Äì move the caret by one character
  • Ctrl+Up / Ctrl+Down ‚Äì move the caret by one visual line inside the multi-line prompt
  • Meta+‚Üê/‚Üí or Ctrl+‚Üê/‚Üí and Meta+b / Meta+f ‚Äì word-wise movement
  • Ctrl-A / Ctrl-E ‚Äì beginning / end of the current line; Ctrl+Home / Ctrl+End ‚Äì beginning / end of the entire prompt
  • Backspace ‚Äì delete the previous character
  • Ctrl-K / Ctrl-U / Ctrl-W or Meta+Backspace ‚Äì kill to EOL / BOL / previous word; Ctrl-Y ‚Äì yank the last killed text
  • Meta+v (or Alt+s) ‚Äì toggle a selection anchor; Ctrl-C / Ctrl-X operate on the active selection (copy / cut)
  • Up / Down arrows, PageUp / PageDown, Home / End ‚Äì scroll conversation history while keeping the editor in Insert mode; this disables Chat_tui.Model.t.auto_follow until the viewport reaches the bottom again
  • Meta+Enter ‚Äì submit the current prompt (Submit_input)
  • ESC ‚Äì switch to Normal mode when pressed without modifiers; when a modifier is present it bubbles up as Cancel_or_quit

The set of bindings is intentionally conservative; unsupported keys are returned as Unhandled so that outer layers may implement fallbacks.

Reactions

All controller variants share the same reaction type (defined in Chat_tui.Controller_types):

| Redraw         ‚Äì visible state changed; re-render the UI
| Submit_input   ‚Äì draft is ready; send it to the assistant
| Cancel_or_quit ‚Äì ESC; cancel streaming or quit if idle
| Compact_context ‚Äì user requested context compaction
| Quit           ‚Äì immediate termination (Ctrl-C / 'q')
| Unhandled      ‚Äì event not recognised; try other handlers

Performance: all operations run in O(length input) at worst (duplicate line); typical keystrokes are sub-millisecond.

type reaction = Controller_types.reaction =
  1. | Redraw
    (*

    The event modified the visible state ‚Äì caller should refresh.

    *)
  2. | Submit_input
    (*

    User pressed Meta+Enter to submit the prompt.

    *)
  3. | Cancel_or_quit
    (*

    ESC request ‚Äì cancel a running request via Eio.Switch.fail when a stream is in flight, or fall back to Quit when idle.

    *)
  4. | Compact_context
    (*

    Trigger conversation compaction via Context_compaction.Compactor ‚Äì the caller should summarise the earlier history, replace elided messages with the summary and then issue Redraw.

    *)
  5. | Quit
    (*

    Immediate quit (Ctrl-C / q). The main loop should terminate the Notty session, release resources and exit.

    *)
  6. | Unhandled
    (*

    Controller didn‚Äôt deal with the event ‚Äì propagate it to higher-level handlers or ignore it.

    *)
val handle_key : model:Model.t -> term:Notty_eio.Term.t -> Notty.Unescape.event -> reaction

handle_key ~model ~term ev is the single public function of the controller hierarchy. It examines model.mode and forwards ev to the appropriate key-map ‚Äì Insert (local), Normal or Cmdline ‚Äì then returns the resulting reaction.

Parameters

  • parameter model

    The mutable snapshot of the UI state that will be modified in-place. Only the record fields are changed ‚Äì no network or disk IO happens here.

  • parameter term

    The Notty terminal abstraction used to query run-time geometry with Notty_eio.Term.size. The value is never modified.

  • parameter ev

    The raw event received from Notty.Unescape.event.

Return value

A Controller_types.reaction. The caller must pattern-match on the result and perform the side-effects described by the variant:

  • Redraw ‚Äì re-render the viewport
  • Submit_input ‚Äì wrap the draft into an OpenAI request and append a pending entry to the history
  • Cancel_or_quit ‚Äì either cancel an in-flight stream or quit when idle
  • Compact_context ‚Äì asynchronously trigger context compaction
  • Quit ‚Äì stop the application immediately
  • Unhandled ‚Äì fall back to global shortcuts or ignore the event

Example

Dispatch an event inside the main loop:

  let reaction = Chat_tui.Controller.handle_key ~model ~term ev in
  match reaction with
  | Redraw -> Renderer.draw ~model ~term
  | Submit_input -> send_prompt_to_assistant model
  | Cancel_or_quit -> handle_escape model
  | Quit -> exit 0
  | Unhandled -> ()
  | Compact_context -> Context_compaction.request ()
