Module Chat_tui.Cmd

Elm-style command interpreter.

Chat_tui separates pure state transitions from impure effects. The controller layer produces values of Chat_tui.Types.cmd which describe what needs to happen (persist the transcript, start a network request, â€¦) without performing the effect. This module is the single implementation point that turns those descriptions into real IO by executing the closures carried by each constructor.

At the time of writing the command set is still small â€“ we only persist the current session and manage the lifetime of a single streaming request â€“ but additional constructors will be added as the UI grows. No other module should perform side-effects directly.

val run : Types.cmd -> unit

The exact implementation of each thunk is opaque to this module and may evolve independently.

run_all cmds executes every command in cmds from left to right using run. It is a thin wrapper around List.iter and provides symmetry with the controller functions that return lists of commands.
